# AxAshell 使用指南

## 适用范围

AxAshell 适合这几类场景：

- 在一个桌面工作区里同时管理本地终端和多个 SSH 会话
- 打开 SSH 会话后直接浏览远程目录、上传下载文件或编辑远程文件
- 把常用连接、外观和快捷键保存在本地配置里
- 通过 WebDAV 或 S3 在多台设备之间同步 SSH 会话配置

## 工作区结构

主界面围绕一个终端工作区展开，常见区域包括：

- 左侧会话区：打开本地终端、选择已保存 SSH 会话、新建 SSH 连接
- 中央终端区：显示本地或远程终端，可多标签和多 Pane 并行使用
- 远程文件区：在 SSH 会话下打开 SFTP 面板，浏览和操作远程文件
- 传输记录：查看下载、上传、暂停、恢复、失败和完成状态
- 设置页：管理外观、字体、快捷键、同步、代理和 X11

## 本地终端与 SSH

### 打开本地终端

启动应用后，可直接在左侧打开本地终端标签页。

### 新建 SSH 会话

1. 打开 `New SSH` 或会话选择器。
2. 输入主机、端口、用户名。
3. 选择密码认证或私钥认证。
4. 需要时填写单会话代理信息。
5. 使用 `Save` 或 `Save & Connect` 保存。

当前 SSH 会话支持：

- 密码认证
- 私钥路径或内联私钥内容
- 可选私钥口令
- 会话级代理设置
- 最近使用时间保存

已保存会话默认写入：

```text
~/.config/ax_ashell/sessions.json
```

## SFTP 文件管理

在 SSH 会话下可以打开 SFTP 面板。当前工作流包括：

- 浏览远程目录
- 显示或隐藏隐藏文件
- 上传文件或文件夹
- 下载文件或文件夹
- 新建文件夹
- 删除选中项
- 用系统默认编辑器打开远程文件，保存后自动上传

传输任务会进入传输记录面板，支持：

- 暂停
- 恢复
- 取消
- 查看完成、失败或中断状态

历史记录最多保留 100 条。

## 多 Pane 与快捷键

AxAshell 把工作区操作做成了可配置快捷键。默认主修饰键为：

- macOS：`Cmd`
- Linux / Windows：`Ctrl`

默认快捷键包括：

- 打开设置：`Cmd/Ctrl + ,`
- 打开会话选择器：`Cmd/Ctrl + O`
- 新建 SSH：`Cmd/Ctrl + N`
- 打开传输记录：`Cmd/Ctrl + T`
- 搜索：`Cmd/Ctrl + F`
- 切换侧边栏：`Cmd/Ctrl + S`
- 聚焦相邻 Pane：`Cmd/Ctrl + H/J/K/L`
- 拆分当前 Pane：`Cmd/Ctrl + Shift + H/J/K/L`
- 关闭当前 Pane：`Cmd/Ctrl + W`
- 复制 / 粘贴：
  - macOS：`Cmd + C` / `Cmd + V`
  - Linux / Windows：`Ctrl + Shift + C` / `Ctrl + Shift + V`

所有这些绑定都可以在设置页的 `Key Bindings` 中改写。

## 设置项

### 外观与字体

当前设置页支持：

- 跟随系统或手动切换明暗主题
- 选择浅色主题和深色主题
- 修改界面字体和终端字体
- 调整界面字号和终端字号
- 调整光标样式
- 选择标题栏样式
- 自定义主题色、背景色、字体亮度和自定义主题名称

### 系统监控与布局

可配置项包括：

- 是否显示系统监控仪表盘
- 监控显示在底部或侧边栏
- 是否锁定窗口布局
- 右键复制粘贴
- 终端关键词高亮

### 配置同步

配置同步支持两种后端：

- WebDAV
- S3 兼容对象存储

同步行为和边界：

- 上传内容会先在本地加密，再上传
- 下载会替换本机保存的 SSH 会话列表
- WebDAV 密码、S3 密钥和加密口令只保留在当前进程内，不写入本地配置文件
- 本地仅保存连接参数，例如 WebDAV 地址、用户名、S3 endpoint、bucket 和对象路径

默认对象名为：

```text
ax_ashell-sync.json
```

### 代理

代理设置支持：

- 启用或禁用全局代理
- 启动时读取环境变量代理
- 选择 `socks5` 或 `http` 代理
- 配置代理地址、端口、用户名和密码

如果启用了“启动时读取环境变量”，应用会优先读取 `ALL_PROXY`、`HTTPS_PROXY`、`HTTP_PROXY` 及其小写形式。

### X11 转发

X11 转发用于 SSH 图形程序场景，当前支持边界如下：

- macOS：依赖 XQuartz
- Windows：依赖 VcXsrv 或 Xming
- Linux / Wayland：依赖本机 `DISPLAY` 或 Xwayland

使用前需要同时满足：

- 本地 X server 可用
- 远端 `sshd` 允许 `X11Forwarding yes`
- 远端 GUI 程序本身支持 X11

## 日志与本地文件

运行日志默认写入：

```text
~/.config/ax_ashell/log
```

SFTP 远程编辑和临时压缩流程会使用配置目录下的临时目录。
